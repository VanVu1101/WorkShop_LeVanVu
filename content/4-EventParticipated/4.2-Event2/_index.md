# Summary Report: “FCAJ Community Day”

### Event Objectives

- Learn modern software architecture and cloud-native development on AWS.
- Improve AWS practical skills through gamified learning platforms and hands-on laboratories.
- Explore real-world software development experiences from Hackathon teams.
- Discover practical applications of AI, DevOps, and productivity techniques in modern software engineering.
- Gain knowledge from experienced speakers working in cloud computing, artificial intelligence, and software development.

### Speakers

- **Huynh Thai Linh** – FCAJ Community
- **Huynh An Khuong** – The Ballers Team
- **Mai Quoc Anh** – The Ballers Team
- **Nguyen Tran Minh Quan** – The Ballers Team
- **Nghia Tran** – AI Project Presenter
- **Tran Minh Quan** – FCAJ Sharing Session

### Key Highlights

#### 1. Level Up Your AWS Skills with Cloud Quest and Floci
**Speaker:** Huynh Thai Linh

The session introduced **AWS Cloud Quest**, a gamified learning platform that allows learners to practice AWS services through interactive missions such as creating EC2 instances, S3 buckets, IAM users, and VPC networks. The speaker also demonstrated **Floci**, a hands-on laboratory platform that provides sandbox environments for safely practicing AWS services.

The presentation emphasized that combining theoretical learning with continuous hands-on practice is one of the most effective ways to master AWS and prepare for certifications such as AWS Certified Cloud Practitioner and AWS Certified Solutions Architect – Associate.

---

#### 2. GenAI-powered App-DB Modernization
**Speakers:** Jignesh Shah, Erica Liu, Fabrianne Effendi

The speakers shared modern application modernization strategies using microservices, Domain-Driven Design (DDD), Event-Driven Architecture, and serverless computing. They explained how organizations can gradually migrate legacy applications while improving scalability, reliability, and operational efficiency.

The session also introduced Amazon Q Developer as an AI assistant capable of supporting software development throughout the SDLC, including code transformation, modernization, testing, and maintenance.

---

#### 3. FCAJ Hackathon Experience
**Presenters:** Huynh An Khuong, Mai Quoc Anh, Nguyen Tran Minh Quan

Members of **The Ballers** shared their Hackathon journey, describing both the excitement and challenges of building a complete software project under strict time constraints. They discussed teamwork, bug fixing, project management, and presentation preparation.

One of the most inspiring messages was that many highly skilled developers still actively participate in hackathons to challenge themselves, exchange knowledge, and continue learning from others. The session highlighted that hackathons are not only competitions but also valuable opportunities for personal growth and collaboration.

---

#### 4. AI-powered Vietnamese Astrology – "Tử vi Đại Việt"
**Presenter:** Nghia Tran

The project demonstrated how artificial intelligence can be combined with traditional Vietnamese astrology to provide personality analysis, strengths, weaknesses, and self-understanding insights.

Using AI-generated analysis together with expert verification, the project showed an innovative way of applying modern technology to preserve cultural knowledge while creating personalized digital experiences.

---

#### 5. The Hidden Iceberg of a Project: DevOps Before Disaster
**Presenter:** Tran Minh Quan

The presentation explained that successful software projects depend not only on application features but also on the hidden infrastructure beneath them. Topics such as CI/CD pipelines, monitoring, logging, security, backup strategies, and infrastructure management were presented as critical foundations for system reliability.

The speaker emphasized that implementing DevOps from the beginning of a project can prevent many production failures before they become serious incidents.

---

#### 6. The Iceberg of Procrastination

This motivational session explored the hidden psychological causes behind procrastination. Rather than being caused by laziness alone, procrastination often originates from fear of failure, perfectionism, anxiety, lack of confidence, and uncertainty.

Several practical techniques were introduced, including breaking tasks into smaller steps, starting with simple actions, accepting imperfect first attempts, and using focused work sessions to build momentum and improve productivity.

### Key Takeaways

- Hands-on practice is essential for developing AWS and cloud computing skills.
- Modern software architecture should combine microservices, event-driven communication, and serverless technologies.
- AI tools can significantly improve productivity when used responsibly throughout the software development lifecycle.
- DevOps practices such as CI/CD, monitoring, and infrastructure automation are critical for building reliable systems.
- Hackathons provide valuable opportunities to collaborate with talented developers, solve real-world problems, and continuously improve technical skills.
- Productivity depends not only on technical knowledge but also on understanding personal habits and psychological barriers.

### Applying to Work

- Continue practicing AWS through Cloud Quest and Floci laboratories.
- Apply DDD, event-driven architecture, and serverless technologies in future projects.
- Integrate AI-assisted development tools to improve software quality and development efficiency.
- Follow DevOps best practices, including automation, monitoring, and backup strategies.
- Participate in more hackathons to strengthen teamwork, communication, and problem-solving skills.
- Apply productivity techniques to improve learning consistency and task management.

### Event Experience

This technical sharing event provided a broad overview of modern cloud computing, artificial intelligence, software architecture, DevOps, and personal productivity. I had the opportunity to learn from experienced engineers and developers who shared both technical knowledge and practical industry experience.

Among all sessions, I was particularly impressed by the Hackathon presentation because it showed that even highly skilled developers continue participating in competitions to challenge themselves and learn from others. I also gained valuable insights into AWS practical learning, modern software architecture, DevOps, and AI-powered application development.

Overall, the event strengthened my technical knowledge, broadened my understanding of current software engineering practices, and motivated me to continue learning, practicing, and participating in future technical communities.

#### Lessons Learned

- Continuous learning and practical experience are essential for professional growth.
- Strong technical skills should always be supported by teamwork and effective communication.
- Modern software development requires knowledge of cloud computing, AI, DevOps, and software architecture.
- Learning from experienced professionals and talented peers is one of the fastest ways to improve technical abilities.
